告别笔记焦虑：我用Obsidian + GitHub + AI 打造的“永久”知识库

发布于 2025-12-21
分类: 效率工具

我用过无数款笔记工具，语雀，OneNote， Notion……，最终我选择了 Obsidian。

为什么选择Obsidian？归根结底是因为“安全感”。

Obsidian 里所有的笔记，本质上都是躺在你本地硬盘里的一个个 Markdown (.md) 文件。哪怕有一天开发商跑路了，你的数据依然完好无损，换个编辑器就能接着用。这种把知识紧紧攥在自己手里的感觉，是任何云端软件都给不了的。

最近，我参考了一套被称为“邪修”的 Obsidian 进阶用法，彻底解决了云同步收费、AI 集成生硬以及图片管理混乱这三大痛点。今天就把这套完全免费、高度自动化的方案分享给大家。

一、 拒绝订阅制：用GitHub 实现终身免费的“无限”云同步

Obsidian 官方的同步服务虽然好用，但每月的订阅费对很多人来说也是一笔开销。其实，对于我们这种把笔记存在本地的用户来说，GitHub 就是最完美的云端保险箱。

我的方案核心：
  1. Git 版本控制：不仅是备份，Git 记录了你每一次的修改历史。误删了某段话？直接回滚到昨天的版本，这种“后悔药”是普通网盘做不到的。
  2. Obsidian Git 插件：你不需要懂复杂的命令行。安装这个插件后，设置好 Auto commit（比如停止编辑1分钟后自动提交），一切都在后台静默完成。你在电脑上写完，它自动推送到 GitHub 私有仓库，完全无感。

这套方案不仅省下了每年的同步费，更重要的是，GitHub 作为计算机基础设施级别的平台，其稳定性远超一般的创业公司网盘。

二、 AI 不止是陪聊：让Gemini 直接“接管”你的本地文件

很多笔记软件的 AI 功能只是在侧边栏挂一个对话框，和你当前的笔记内容是割裂的。但在 Obsidian 的这套玩法里，AI 是真正融入文件系统的。

我使用了 Google 的 Gemini CLI 工具（配合 Node.js），它最大的优势是能直接读取和操作本地文件。

我是怎么用的？
  • 模仿文风写作：我让 AI 读取我过去三年的所有文章脚本，分析我的语气、用词习惯。然后，我只需给它一个新主题，它就能生成一份带有我强烈个人风格的初稿。
  • 批量整理：面对一堆杂乱的选题，我让 AI 自动创建文件夹、分类归档，甚至帮我把旧笔记里的灵感提取出来，生成新的选题大纲。

这种直接对本地 .md 文件进行批量“手术”的能力，才是 AI 笔记的终极形态。

三、 细节控的强迫症解药：图片与多端同步

用 Markdown 写笔记，最头疼的就是图片。以前直接粘贴，图片会乱飞，或者链接格式不标准，导致在别的地方打不开。

我的解决方案：
  • Custom Attachment Location 插件：强制把粘贴的图片自动存入当前笔记同级的assets文件夹。
  • 标准化路径：配置插件将图片链接转换为标准的 Markdown 语法。这样，我的笔记推送到 GitHub 后，直接在网页端打开也能完美显示图片，甚至用 VS Code 打开也能正常预览。
📱 手机端怎么办？ 即使是安卓手机，配合 GitHub 也能实现同步。将电脑端的笔记库复制到手机，配置好 Git 和 Token，出门在外产生的灵感，写完自动推送到云端，回到家电脑上一拉取，无缝衔接。

写在最后

这套方案被戏称为“邪修”，因为它确实有一点门槛（比如需要注册 GitHub、安装 Node.js）。但一旦配置完成，你将获得一个：

  • 完全免费
  • 数据绝对私有
  • 拥有AI 生产力
  • 全平台同步

的终极知识库。在这个数据并不总是属于你的时代，愿我们都能构建属于自己的、永不丢失的数字花园。
